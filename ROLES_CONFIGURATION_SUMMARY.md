### Где добавлять переменные

#### 🔸 Replit (рекомендуется для быстрого старта)
1. Откройте проект
2. Нажмите иконку "Secrets" (замок) слева
3. Добавьте каждую переменную

1. Settings → Config Vars
2. Добавьте переменные

1. Создайте `.env` файл в корне проекта
2. Вставьте переменные
3. Убедитесь, что `.env` в `.gitignore`

## 🔄 Приоритет Проверки Прав

Система проверяет права в следующем порядке (от наивысшего к наименьшему):

1. **Овнер сервера** → ВСЕ ПРАВА (всегда)
2. **Administrator permission** → ВСЕ ПРАВА (всегда)
3. **ADMIN_ROLE_IDS** → ВСЕ ПРАВА
4. **GM_ROLE_IDS** → AP, SP, деньги, предметы
5. **GIVE_STYLE_ROLE_IDS** → только стили

---

## ✅ Как Проверить, Что Всё Работает

1. **Перезагрузите бота** после добавления переменных

2. **На Discord сервере:**
   - Назначьте роли пользователям
   - Убедитесь, что роли имеют нужные ID

3. **Тестирование:**
   - Пользователь с ADMIN_ROLE_IDS должен использовать ВСЕ команды
   - Пользователь с GM_ROLE_IDS не должен видеть команды окроме AP/SP/деньги/предметы
   - Пользователи с GIVE_STYLE_ROLE_IDS должны иметь только доступ к стилям

4. **Если команда не работает:**
   - Проверьте консоль на ошибки
   - Убедитесь, что ID роли скопирован правильно
   - Проверьте, что переменная окружения сохранена и прочитана

## 📞 Быстрая Справка

**Нужно дать админу ВСЕ права?**
→ Добавьте его роль в `ADMIN_ROLE_IDS`

**Нужно дать только управление AP/SP/деньги/предметы?**
→ Добавьте роль в `GM_ROLE_IDS`

**Нужно дать только выдачу стилей?**
→ Добавьте роль в `GIVE_STYLE_ROLE_IDS`

**Как получить ID роли?**
→ Developer Mode → правый клик на роль → Copy Role ID

**Где скинуть переменные?**
→ Посмотрите "Где добавлять переменные" выше
