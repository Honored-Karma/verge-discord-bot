# Environment Variables Configuration Guide

## Обязательные переменные

### Discord Bot Token
```
DISCORD_TOKEN=your_bot_token_here
```
Токен вашего Discord бота, полученный через Developer Portal.

### MongoDB Connection
```
MONGODB_URI=mongodb://localhost:27017/discord-rpg-bot
```
Строка подключения к MongoDB.

## Пример полной конфигурации для хостинга

```env
# Discord Bot Configuration
DISCORD_TOKEN=your_actual_bot_token_here
MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/discord-rpg-bot

# Admin Roles (non-owner admins with all permissions)
ADMIN_ROLE_IDS=1438987035862892544

# Style Giver Roles (Анкетологи - can give styles)
GIVE_STYLE_ROLE_IDS=1431759459528610013,1438987088975626251

# Game Master Roles (can manage AP, SP, currency, items)
GM_ROLE_IDS=1431759694179074162,1438987072026185808
```

## Как получить ID роли в Discord

1. Включите **Developer Mode** в Discord (User Settings → App Settings → Advanced → Developer Mode)
2. Щелкните правой кнопкой мыши на роль в Discord сервере
3. Выберите **Copy Role ID**
4. Вставьте ID в переменную окружения

---

## Приоритет проверки прав доступа

1. **Овнер сервера** - всегда имеет все права
2. **Пользователи с Administrator permission** - всегда имеют все права
3. **ADMIN_ROLE_IDS** - роли, указанные в этой переменной, имеют все права администратора
4. **GM_ROLE_IDS** - роли для управления AP, SP, деньгами и предметами
5. **GIVE_STYLE_ROLE_IDS** - роли для выдачи боевых стилей

## Как закинуть переменные на хостинг

### Если вы используете Replit
1. Откройте проект в Replit
2. Нажмите на иконку "Secrets" (замок) на левой панели
3. Добавьте каждую переменную:
   - Key: `DISCORD_TOKEN`, Value: ваш токен
   - Key: `MONGODB_URI`, Value: ваша строка подключения
   - Key: `ADMIN_ROLE_IDS`, Value: `1438987035862892544`
   - Key: `GIVE_STYLE_ROLE_IDS`, Value: `1431759459528610013,1438987088975626251`
   - Key: `GM_ROLE_IDS`, Value: `1431759694179074162,1438987072026185808`

### Если вы используете другой хостинг (Heroku, Railway, Render и т.д.)
1. Найдите раздел "Environment Variables" или "Secrets" в панели управления хостингом
2. Добавьте все переменные по одной
3. Перезагрузите приложение

### Если вы используете .env файл локально
1. Скопируйте содержимое конфигурации выше
2. Создайте файл `.env` в корне проекта
3. Вставьте переменные
4. Убедитесь, что `.env` добавлен в `.gitignore` (для безопасности)

## Возможные проблемы и решения

### Проблема: Команда не работает, хотя роль назначена
- Проверьте, что ID роли скопирован правильно (без пробелов)
- Убедитесь, что переменная окружения сохранена
- Перезагрузите бота

### Проблема: Переменные не распознаются
- Убедитесь, что нет опечаток в названиях переменных
- Проверьте, что используются точные имена:
  - `ADMIN_ROLE_IDS`
  - `GIVE_STYLE_ROLE_IDS`
  - `GM_ROLE_IDS`

### Проблема: Администраторы не имеют доступа
- Проверьте, что пользователь имеет нужную роль
- Убедитесь, что роль не скрыта
- Проверьте правильность ID роли
