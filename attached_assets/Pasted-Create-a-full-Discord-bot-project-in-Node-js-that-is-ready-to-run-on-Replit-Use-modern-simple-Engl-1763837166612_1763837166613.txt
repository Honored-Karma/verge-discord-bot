Functional rules (implement exactly):
1. AP rules:
   - Social RP: gives 20 AP, cooldown 12 hours per player.
   - Training (command `/train`): requires `text` argument of at least 800 characters (count spaces). Gives 10 AP, cooldown 5 hours.
   - Every 100 AP the player can create or learn a new technique. (store as a note in player or items; a simple `techniques` text field or separate table optional).
   - Reaching **1000 AP** flags player as eligible for Avatar/Embodiment (unlocked_avatar flag). GM/admin must confirm via `/set-ap` or a special admin command to mark final awakening.
   - Enforce cooldowns and store last_ap_timestamp in players.

2. SP rules:
   - SP is style-specific. Each style has its own SP counter in `player_sp`.
   - Training SP awarding: admins can use `/set-sp` to modify SP for style (no automatic battle SP as requested).
   - There are style mastery thresholds: 0 (Novice), 350 (Owner), 1000 (Expert), 2500 (Master). Show rank based on SP for that style.

3. Styles:
   - Style list default (create these rows on DB init): 
     - "Aikido: reverse"
     - "Blood Taekwondo"
     - "Muay Thai"
     - "Dark Jiu-Jitsu"
     - "Sun Kendo"
     - "Qi boxing"
     - "Wolgwang Sword Style"
     - "Kyokushin Karate"
   - Only admins (role "Game Master" or an `ADMIN_ID` from env) can add new styles (`/add-style`) or remove them. `/styles-list` lists all styles and shows total players with SP in each.

4. Inventory:
   - Admins can add items with `/give-item @user item_id` which will show in inventory.
   - Users can view `/inventory` and use `/use-item item_id` (implement `/use-item` only if you want; at minimum display use action in embed).
   - Item effects can be simple: `{"ap":50}` or `{"sp":{"style":"Muay Thai","value":30}}`. When an item is used, apply those effects and write action to admin_actions log.

5. Profiles & Displays:
   - `/register` registers user in DB (id, username).
   - `/profile @user` shows embed with:
     - Username & avatar
     - AP (value) and progress bar to next 100 AP (use progress bar util)
     - List of styles with SP per style and rank label (Novice/Owner/Expert/Master)
     - Inventory summary (first 5 items)
     - Avatar/unlocked flag
     - Footer: user ID + last AP timestamp in human readable
   - Progress bar: e.g. 20 blocks wide, show filled and empty blocks and percentage (text, not images).

6. Commands (slash names and arguments — implement in `deploy-commands.js` and `commands/`):
   - `/register` — register yourself (no args).
   - `/train text:<string>` — submits training text; validate ≥800 chars; award 10 AP; send embed with new AP and progress bar; store timestamp for cooldown 5h.
   - `/social-rp text:<string>` — submit social rp; award 20 AP; cooldown 12h.
   - `/profile user:<user(optional)>` — show profile embed.
   - `/styles-list` — show styles and short stats (players count per style).
   - `/add-style name:<string> description:<string>` — admin only, adds style to DB.
   - `/set-ap user:<user> amount:<integer>` — admin only, set AP to absolute value.
   - `/add-ap user:<user> amount:<integer>` — admin only, add AP (delta).
   - `/set-sp user:<user> style:<style_name> amount:<integer>` — admin only, set SP for a style.
   - `/add-sp user:<user> style:<style_name> amount:<integer>` — admin only, add SP.
   - `/inventory user:<user(optional)>` — show inventory items.
   - `/give-item user:<user> item_id:<string> qty:<integer>` — admin only.
   - `/leaderboard sort_by:<choice(ap|sp|balance)> limit:<integer optional>` — show top players. For `sp` show total SP across styles.

7. Admin checks:
   - Use either guild role name `Game Master` OR `ADMIN_IDS` env variable with comma-separated admin user ids. If neither found, fallback to guild Owner. All admin-only commands must check this.

8. Cooldowns & anti-farm:
   - Store last action timestamps for `/train` and `/social-rp` individually.
   - If a player tries during cooldown, return human readable cooldown left (hours/mins).
   - For `/train`, check text for "not mostly links": require at least 50% letters/digits or spaces. Reject otherwise.

9. Progress & rank text:
   - For AP progress, show "Progress to next Technique" where each technique is unlocked per 100 AP.
   - For style rank: compute rank by SP thresholds and show color indicator in embed.

10. Embeds & UX:
    - Use neat embeds with fields and inline fields.
    - Use emoji icons when possible (but keep text fallback).
    - Example embed for `/profile`: Title = "Profile — <username>", fields: AP (value + progress bar), Styles (inline listing of style: SP (rank)), Inventory (small list), Footer = "ID: ... • Last AP: ...".

11. DB helpers and data manager:
    - Provide `db.js` to initialize DB and run `init.sql` if tables missing.
    - Provide `dataManager.js` with functions: getPlayer(id), createPlayer(id, username), addAP(id, amount), setAP, getSP(id, styleId), addSP, listStyles(), addStyle(name, desc, adminId).
    - Wrap DB calls in try/catch and log admin actions in `admin_actions` table.

12. Logging:
    - Every admin change writes row to `admin_actions` table and appends a line to `/logs/actions.log`.

13. README:
    - Explain how to set secrets: TOKEN, CLIENT_ID, GUILD_ID (for dev), ADMIN_IDS (optional), and how to run `node deploy-commands.js` then `node index.js`.
    - Explain how to edit SQL or switch to production DB.
