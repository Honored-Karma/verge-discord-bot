ШАГИ ДЛЯ REPLIT (рекомендуется):
─────────────────────────────────────
1. Откройте ваш проект на Replit.com
2. На левой панели нажмите иконку 🔒 "Secrets"
3. Нажмите "Add new secret"
4. Добавьте переменные:

   Переменная 1:
   Key: ADMIN_ROLE_IDS
   Value: 1438987035862892544

   Переменная 2:
   Key: GIVE_STYLE_ROLE_IDS
   Value: 1431759459528610013,1438987088975626251

   Переменная 3:
   Key: GM_ROLE_IDS
   Value: 1431759694179074162,1438987072026185808

5. Не забудьте добавить DISCORD_TOKEN и MONGODB_URI!
6. Нажмите "Run" для перезагрузки бота


ШАГИ ДЛЯ HEROKU:
─────────────────────────────────────
1. Откройте панель управления Heroku
2. Settings → Config Vars
3. Нажмите "Reveal Config Vars"
4. Нажмите "Add"
5. Добавьте все переменные по одной
6. Дайте приложению время на перезагрузку


ШАГИ ДЛЯ RAILWAY / RENDER / ДРУГИХ:
─────────────────────────────────────
1. Откройте панель хостинга
2. Найдите раздел "Environment Variables" или "Environment"
3. Добавьте каждую переменную
4. Нажмите "Deploy" или "Save"

✅ ПРОВЕРКА РАБОТОСПОСОБНОСТИ
═══════════════════════════════════════════════════════════════════════════════════

После добавления переменных:

1. Перезагрузите бота
2. В Discord сервере назначьте нужные роли пользователям
3. Пользователь с ADMIN_ROLE_IDS должен видеть ВСЕ команды администратора
4. Пользователи с GIVE_STYLE_ROLE_IDS должны видеть ТОЛЬКО give-style команды
5. Пользователи с GM_ROLE_IDS должны видеть ТОЛЬКО AP/SP/деньги/предметы команды

Если команда не работает:
→ Проверьте консоль на ошибки
→ Убедитесь, что ID роли скопирован правильно (БЕЗ ПРОБЕЛОВ)
→ Перезагрузите бота
→ Проверьте, что роли назначены правильным пользователям

🔐 ПРИОРИТЕТ ПРОВЕРКИ ПРАВ
═══════════════════════════════════════════════════════════════════════════════════

Система проверяет права в этом порядке:

1. Овнер сервера
   → ✅ ВСЕ ПРАВА (всегда)

2. Пользователь с Administrator permission
   → ✅ ВСЕ ПРАВА (всегда)

3. Роль в ADMIN_ROLE_IDS
   → ✅ ВСЕ ПРАВА

4. Роль в GM_ROLE_IDS
   → ✅ AP, SP, деньги, предметы

5. Роль в GIVE_STYLE_ROLE_IDS
   → ✅ Только стили

Остальные пользователи
   → ❌ Доступ запрещен
